Question-1: What are the different ways to select an element in the DOM?

JavaScript provides several ways to select DOM elements. "getElementById" selects an element by ID, while getElementsByClassName and "getElementsByTagName" return collections based on class or tag name. "querySelector" selects the first matching element using a CSS selector, and "querySelectorAll" returns all matching elements as a NodeList.

Question-2: What is the difference between innerHTML, innerText, and textContent ?

"innerHTML", "innerText", and "textContent" are used to get or set content in a DOM element, but they behave differently. "innerHTML" returns or sets the HTML content, including any tags, which allows parsing of HTML but can pose security risks if handling user input. "innerText" retrieves or sets only the visible text, ignoring hidden elements and extra spaces, and it also triggers reflow, which may impact performance. "textContent" returns or sets all text content, including hidden elements, while preserving spacing, making it faster and more secure than "innerText" since it does not parse HTML.

Question-3: What is event delegation in the DOM?

Event delegation is a technique where a parent element handles events for its child elements using event bubbling. Instead of adding event listeners to each child, a single listener on the parent detects events from dynamically added elements, improving performance.

Question-4: What is event bubbling in the DOM?

Event bubbling is a DOM mechanism where an event starts from the target element and propagates **upward** through its ancestors. For example, if a button inside a div is clicked, the event first triggers on the button, then bubbles up to the div, and further up to the document. This allows parent elements to handle events from their children, enabling techniques like event delegation.
